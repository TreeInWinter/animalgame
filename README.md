# animalgame

一个面向 **2 岁左右儿童** 的纯前端动物认知游戏（无后端，数据存储在 `localStorage`）。

## 当前版本（v1.0）

已具备：

- 动物认知卡片（点击播放音效）
- 动物声音匹配测验（单题 3 选 1）
- 本地进度记录与重置

## 目标用户

- 主要用户：2~4 岁儿童
- 辅助用户：家长（陪玩、查看学习进展）

## 迭代路线图（可直接按此开发）

---

### v1.1 交互与可用性增强（优先级：高）

**目标**：降低幼儿误触成本，提升反馈即时性。

功能项：

1. 大按钮模式（移动端优先，最小可点区域 >= 56px）
2. 卡片点击视觉反馈（缩放/高亮）
3. 答题后正误反馈增强（颜色 + 动画 + 声音）
4. “再听一次”按钮（当前题目音效可重复播放）
5. 首屏引导（1 句话告诉孩子怎么开始）

验收标准：

- 手机竖屏下无需横向滚动
- 连续 10 次点击不出现无响应
- 答题后 300ms 内有明确反馈

---

### v1.2 学习机制升级（优先级：高）

**目标**：从“随机题”升级为“可学习、可复习”的轻量流程。

功能项：

1. 难度分级：
   - 简单：2 选 1
   - 普通：3 选 1
   - 进阶：4 选 1
2. 连对奖励（例如连对 3 次出现鼓励文案）
3. 错题回放（最近错题优先再次出现）
4. 每日学习次数统计（按日期记录）

验收标准：

- 可以切换难度并即时生效
- 错题在后续 3~5 题内有更高概率重现
- 刷新页面后统计数据不丢失

---

### v1.3 家长模式（优先级：中）

**目标**：让家长看到孩子学习进展，支持简单内容管理。

功能项：

1. 家长入口（长按或角落入口，避免孩子误入）
2. 学习统计面板：
   - 总答题数
   - 正确率
   - 连续学习天数
   - 各动物掌握度
3. 数据重置（带二次确认）
4. 开关控制：背景音乐/音效/动画强度

验收标准：

- 无后端情况下统计可用且稳定
- 重置后所有统计与题目状态清零
- 孩子模式下不出现复杂管理按钮

---

### v1.4 内容扩展（优先级：中）

**目标**：扩展可学习内容，保持结构可持续增长。

功能项：

1. 动物数量从 6 扩展到 20+
2. 分类学习（农场、森林、海洋）
3. 中英文名称切换（可选）
4. 动物数据抽离为独立 `animals.json` 或 `data.js`

验收标准：

- 新增动物无需改动核心逻辑
- 分类切换后题目仅来自当前分类
- 中英文切换不影响测验流程

---

## 推荐实施分工

- **地字一号**：v1.1 + v1.3（交互体验、UI、家长模式）
- **玄字一号**：v1.2 + v1.4（学习机制、数据结构、内容扩展）

## 技术约束

- 继续保持纯前端（HTML/CSS/JS）
- 不引入构建工具也可运行
- 保持 `python3 -m http.server 8000` 即可本地启动

## 本地运行

```bash
python3 -m http.server 8000
```

浏览器打开：`http://localhost:8000`
